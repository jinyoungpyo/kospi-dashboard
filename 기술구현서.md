## 1. 시스템 개요

매일 오후 9시(KST) GitHub Actions가 자동 실행되어 금융투자협회 FreeSIS와 네이버 금융에서 KOSPI 시장 자금 데이터를 수집하고, 단일 JSON 파일로 가공한 뒤 웹 대시보드에 자동 배포하는 시스템이다. 사람이 개입하지 않아도 매 영업일 최신 데이터로 갱신된다.

---

## 2. 시스템 아키텍처

```
[데이터 원천]                [자동화 파이프라인]            [배포·서빙]
                                                    
FreeSIS API  ──┐             GitHub Actions          Cloudflare Pages
               ├──► Python Scripts ──► data.json ──► (GitHub push 감지 →
Naver 금융  ──┘             (src/)                   자동 빌드·배포)
                                                    
                                                         │
                                                         ▼
                                                  웹 브라우저
                                              (HTML + ApexCharts)
```

**데이터 흐름 요약**

1. GitHub Actions 스케줄러가 매일 21:07 KST 실행
2. Python 스크립트 4개가 순서대로 데이터 수집·가공
3. `dashboard/assets/data.json` 갱신
4. git commit → GitHub push
5. Cloudflare Pages가 push를 감지해 자동 배포
6. 사용자는 브라우저에서 최신 대시보드 확인

## 4. 프론트엔드 구조

### 4.1 기술 스택

| 항목 | 내용 |
|------|------|
| 파일 | `dashboard/index.html` (단일 파일) |
| 차트 라이브러리 | ApexCharts 3.54.1 (CDN) |
| 데이터 로드 방식 | `fetch("assets/data.json")` — 서버 API 없음 |
| 빌드 도구 | 없음 (정적 HTML) |
| 외부 의존성 | ApexCharts CDN 1개 |

### 4.2 화면 구성

| 섹션 | 설명 |
|------|------|
| KPI 카드 | 11개 지표 최신값 + 전일 대비 변화율 표시. 클릭 시 해당 지표 차트로 전환 |
| 개별 지표 추이 | 선택 지표의 시계열 차트. 기간 버튼(1M/3M/6M/1Y/전체)으로 범위 조정 |
| 예탁금 vs 거래대금 | D+0/1/2 선택으로 결제 시점 맞춤 이중축 차트 |
| 자금 회전율 | 거래대금/예탁금 비율(%) 시계열 |
| 구간별 상관관계 | 전체·1차상승·조정·2차상승 4구간 Pearson r / Δr 테이블 |
| 30일 롤링 r | KOSPI ↔ 예탁금 동조성 변화 꺾은선 |
| 3대 대기성 자금 분석 | 예탁금·파생예수금·RP매도잔고 × KOSPI 구간별 r/Δr 테이블 + 인사이트 박스 |
| 이상치 탐지 | 30일 롤링 z-score 기준 이상치 목록 (임계값 2.0/2.5/3.0 선택) |

### 4.3 통계 계산

모든 계산은 클라이언트 JavaScript에서 수행 (서버 연산 없음):

- **Pearson 상관계수**: `pearson(xs, ys)` 함수
- **롤링 상관계수**: 30거래일 슬라이딩 윈도우
- **Δr**: 일별 변화량 시계열에 대한 상관계수
- **z-score**: 30일 롤링 평균·표준편차 기준

## 6. 인증 및 권한

이 시스템은 **읽기 전용 공개 대시보드**로, 별도 로그인·인증 없음.

| 항목 | 내용 |
|------|------|
| 접근 제어 | 없음 (공개 URL) |
| 데이터 쓰기 | GitHub Actions 봇 계정만 가능 |
| GitHub 저장소 권한 | `contents: write` (Actions가 commit·push에 필요) |
| 민감 정보 | SMTP 계정 정보는 GitHub Secrets에만 저장 (코드에 노출 없음) |

접근을 제한해야 할 경우 Cloudflare Pages의 Access 기능으로 이메일 인증 추가 가능.

### 7.1 Cloudflare Pages 설정

| 항목 | 값 |
|------|----|
| 프로젝트명 | kospi-dashboard-pages |
| 연결 저장소 | jinyoungpyo/kospi-dashboard (main 브랜치) |

| 배포 디렉터리 | `dashboard/` |
| 트리거 | GitHub main 브랜치 push 시 자동 |

### 7.2 배포 흐름

```
Python 스크립트 실행
       │
data.json 갱신
       │
git commit & push (github-actions[bot])
       │  ※ push 충돌 시 rebase 후 최대 3회 재시도
       │
Cloudflare Pages 자동 감지
       │
1~2분 내 웹사이트 반영
```

### 7.3 GitHub Secrets 목록

| 시크릿 키 | 용도 |
|-----------|------|
| `SMTP_HOST` | 메일 서버 주소 |
| `SMTP_PORT` | 메일 서버 포트 |
| `SMTP_USER` | 발신 계정 |
| `SMTP_PASS` | 발신 계정 비밀번호 |
| `MAIL_TO` | 수신 이메일 주소 |
| `MAIL_FROM` | 발신 이메일 (미설정 시 SMTP_USER 사용) |

## 9. 운영 및 장애 대응

### 9.1 정상 동작 확인

- GitHub Actions → `daily-update` 워크플로 실행 내역 확인
- 대시보드 우측 상단 "최종 데이터" 날짜가 당일(영업일)인지 확인

### 9.2 자주 발생하는 장애 패턴

| 증상 | 원인 | 조치 |
|------|------|------|
| 데이터 날짜가 어제 이전으로 멈춤 | FreeSIS API 응답 없음 또는 형식 변경 | Actions 로그 확인 → `collect_freesis.py` 응답 구조 확인 |
| `collect_naver_*.py` 실패 | 네이버 구조 변경 | `continue-on-error: true` 설정으로 전체 파이프라인은 계속 실행됨 |
| push 충돌 | 동시 실행 (21:07 / 22:07 두 크론) | 워크플로가 rebase 재시도 3회 내장 |
| 워크플로 실패 시 이메일 수신 | 정상 동작 (장애 알림 메일) | Actions 로그에서 실패 단계 확인 |

### 9.3 수동 재실행

GitHub 저장소 → Actions 탭 → `daily-update` → `Run workflow` 버튼으로 즉시 실행 가능.

### 9.4 데이터 범위 확장

연도가 바뀌거나 수집 범위를 늘릴 경우 `build_dashboard_data.py`의 CSV 파일명 패턴과 `collect_freesis.py`의 날짜 범위 파라미터를 수정.

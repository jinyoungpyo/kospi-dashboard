## 1. 업무 영역

### 1.1 목적

KOSPI 시장의 자금 흐름 지표(예탁금·파생예수금·RP매도잔고 등)를 매일 자동으로 수집하고, KOSPI 지수와의 상관관계를 분석하여 시장 국면을 파악하는 내부 분석 대시보드.

### 1.2 대상 사용자

| 역할 | 접근 방식 |
|------|----------|
| 내부 분석 담당자 | 이메일 OTP 인증 후 대시보드 열람 |
| 운영자 | GitHub Actions 수동 실행, 허용 이메일 관리 |

### 1.3 분석 항목

| 분류 | 지표 |
|------|------|
| 시장 현황 | KOSPI 지수, 거래대금 |
| 대기성 자금 | 투자자예탁금, RP매도잔고, 파생상품예수금 |
| 외국인 동향 | 외국인 비중(%), 외국인 순매수(억원) |
| 레버리지 자금 | 신용잔고, 대차잔고, 증권담보융자 |

## 4. 데이터 파이프라인

### 4.1 수집 단계

**실행 주기**: 매일 21:07 KST (보완 실행 22:07 KST)  
**실행 환경**: GitHub Actions (ubuntu-latest, Python 3.11)

| 스크립트 | 데이터 원천 | 수집 항목 | 출력 파일 |
|----------|------------|----------|----------|
| `collect_freesis.py` | FreeSIS API (`STATSCU0100000060`) | 투자자예탁금, 파생예수금, RP매도잔고 | `jeungsi_jageum_2026Q1Q2.csv` |
| `collect_freesis.py` | FreeSIS API (`STATSCU0100000070`) | 신용잔고, 증권담보융자 | `신용공여_잔고_2026Q1Q2.csv` |
| `collect_freesis.py` | FreeSIS API (`STATSCU0100000140`) | 대차잔고 | `대차거래_추이_2026Q1Q2.csv` |
| `collect_freesis.py` | FreeSIS API (`STATSCU0100000020`) | KOSPI지수, 거래대금, 시가총액, 외국인비중 | `유가증권시장_2026Q1Q2.csv` |
| `collect_naver_kospi.py` | 네이버 금융 | 당일 KOSPI 종가 (T+0 보완) | CSV 보완 |
| `collect_naver_investor.py` | 네이버 금융 | 외국인 순매수 | CSV 보완 |

> FreeSIS는 전영업일 마감 기준 데이터를 제공하므로 당일 T+0 데이터는 네이버 금융으로 보완.  
> `collect_naver_*.py`는 `continue-on-error: true` 처리 — 실패해도 전체 파이프라인 계속 실행.

### 4.2 가공 단계

**스크립트**: `src/build_dashboard_data.py`

4개 CSV를 날짜 기준으로 병합하여 단일 JSON 파일 생성.

```json
{
  "range": { "start": "2026-01-02", "end": "2026-06-02" },
  "rows": [
    {
      "date": "2026-01-02",
      "KOSPI지수": 2450.00,
      "투자자예탁금": 62000000,
      "파생예수금": 18000000,
      "RP매도잔고": 95000000,
      "신용잔고": 20000000,
      "증권담보융자": 25000000,
      "대차잔고": 70000000,
      "거래대금": 12000000,
      "외국인_비중_pct": 31.5,
      "외국인_순매수_억": 1200
    }
  ]
}
```

> 모든 금액 필드 단위: **백만원** / 화면 표시 시 1,000,000으로 나눠 조원으로 변환

### 4.3 알림 단계

**스크립트**: `src/check_alerts.py`

| 지표 | 트리거 조건 | 방향 |
|------|-----------|------|
| KOSPI지수 | 전일 대비 −10% 이상 | 하락 |
| 투자자예탁금 | 전일 대비 −10% 이상 | 감소 |
| 거래대금 | 전일 대비 −10% 이상 | 감소 |
| 신용잔고 | 전일 대비 +10% 이상 | 증가 |
| 대차잔고 | 전일 대비 +10% 이상 | 증가 |

조건 충족 시 GitHub Actions output으로 `send=true` 전달 → `dawidd6/action-send-mail@v3`로 이메일 발송.  
워크플로 자체 실패 시에도 별도 실패 알림 이메일 발송.

## 6. 배포 구조

### 6.1 배포 흐름

```

        │
        ▼

        ▼
```

| 항목 | 값 |

### 6.3 수동 재배포

GitHub → Actions 탭 → `daily-update` → `Run workflow`로 즉시 실행 가능.

## 7. 운영 및 장애 대응

### 7.1 정상 동작 확인

- GitHub Actions → `daily-update` 워크플로 최근 실행 결과 확인 (녹색 체크)
- 대시보드 우측 상단 "최종 데이터" 날짜가 최근 영업일과 일치하는지 확인

### 7.2 장애 패턴별 조치

| 증상 | 원인 | 조치 |
|------|------|------|
| 데이터 날짜가 멈춤 | FreeSIS API 무응답 또는 형식 변경 | Actions 로그 확인 → `collect_freesis.py` 점검 |
| 네이버 수집 실패 | 네이버 페이지 구조 변경 | `continue-on-error` 설정으로 전체 실행은 유지됨 |
| push 충돌 | 두 크론(21:07 / 22:07) 동시 실행 | 워크플로 내장 rebase 재시도 3회로 자동 해결 |
| 장애 알림 메일 수신 | 워크플로 실패 | 메일 본문의 Actions Run 링크에서 실패 단계 확인 |
| 대시보드 접속 불가 | Cloudflare Pages 장애 | `cloudflarestatus.com` 확인 |
